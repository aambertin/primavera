(static) @After(…fns)

Source:
  • core.js, line 123

Apply one or more functions AFTER a method is executed. Decorators can transform or choose to return a different return value by just returning something or by altering the attributes of a returned object.

Each function will receive as only parameter the return value of the previous one. If no value is returned, the previous non-undefined return value will be kept.

Examples
@After(() => debug('do nothing'))
function testfunction() { return 'original' }
testfunction() == 'original'
@After((value) => { return 'altered'})
testfunction() { return 'original' }
testfunction() == 'altered'
@After((obj) => { 
    obj.altered = true
})
testfunction() { return {altered:false} }
testfunction().altered == true

(static) @Before(…fns)

Source:
  • core.js, line 60

Apply one or more decorators BEFORE a method is executed. Decorators can transform the original arguments accessing the arguments of the function, or by providing an array as return value which will override the entire arguments chain to be passed to the next intercepting function or the target method.

Examples
@Before(() => debug('do nothing'))
function somefunction() { ... }
@Before((...args) => { return ['altered']})
testfunction(...args) {
    // args[0] == 'altered'
}
@Before((...args) => { 
    const altered = [...args]
    altered[0] = 'altered'
})
testfunction(...args) {
    // args[0] == 'altered'
}

(static) @Bind(…fixtures)

Source:
  • core.js, line 279

Bind a given object element to the decorated method.

Examples
@Bind('myAttribute')
testfunction (value) {
    value == this.myAttribute
}
@Bind('nested.attribute')
testfunction (value) {
    value == this.nested.attribute
}
@Bind('someFunction')
testfunction (value) {
    value == this.someFunction()
}
@Bind('someFunction().nestedAttribute')
testfunction (value) {
    value == this.someFunction().nestedAttribute
}

(static) @SmartPooling()

Source:
  • core.js, line 187

Smart pooling describes the ability to keep a pool of "Singletons" based on their instantiation parameters.

This helps optimize creation/destruction of objects in the container, reducing memory footpring as well as saving resources by reducing object creation operations.

To be used toogether with @Injectable or one of its aliases.

Example
@SmartPooling()
class PooledClass {
    constructor() {
        this.random = Math.random()
    }
}

...
let base = new PooledClass({ value: 1 })
let same = new PooledClass({ value: 1 })
let different = new PooledClass({ value: 2 })

base.random == same.random
base.random == different.random
